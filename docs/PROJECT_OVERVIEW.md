# Что делает проект

`events-backend` превращает неструктурированные Telegram-посты в структурированные события и отдаёт их через FastAPI.

## Цель

Сформировать единый pipeline:

1. собрать посты о событиях из Telegram;
2. извлечь из них структуру (дата, категории, цена, описание и т.д.);
3. сохранить результат в MongoDB;
4. отдать клиентам API с фильтрацией, пагинацией и персонализацией.

## Основные подсистемы

## 1. `src/telegram_parser`

Назначение:
- подключение к Telegram через Telethon;
- парсинг каналов;
- фильтрация по хештегам и датам;
- сохранение исходных постов в `raw_posts`.

Результат:
- база получает сырой поток постов;
- дубликаты по `post_id` и каналу не размножаются.

## 2. `src/event_extraction`

Назначение:
- обработка новых записей из `raw_posts`;
- извлечение событий из текста через LLM;
- дедупликация через Qdrant;
- сохранение в `processed_events`.

Результат:
- на выходе готовые карточки событий для выдачи в API.

## 3. `api` (FastAPI)

Назначение:
- регистрация и логин пользователей;
- выдача списка событий и деталей;
- действия пользователя (`like`, `dislike`, `participate`);
- персонализация по интересам.

Ключевые особенности:
- JWT-авторизация;
- cursor-based pagination;
- фильтры по категориям, цене, датам, поиску.

## 4. `src/monitoring` + `src/webui`

Назначение:
- экспонирование метрик Prometheus;
- диагностический WebUI;
- health endpoint для оркестрации и наблюдаемости.

## Поток данных

1. Telegram -> `raw_posts`
2. Event Extraction -> `processed_events`
3. API читает `processed_events` и `users`
4. Действия пользователя -> `user_actions`, обновление интересов

## Основные коллекции MongoDB

- `raw_posts` — сырые посты из Telegram.
- `processed_posts` — отметки уже обработанных постов.
- `processed_events` — структурированные события для API.
- `users` — пользователи и их интересы.
- `user_actions` — реакции пользователя на события.

## Точки входа

- `run_parser.py` — одноразовый запуск парсинга.
- `run_event_extraction.py` — извлечение событий.
- `run_api.py` — запуск FastAPI.
- `run_webui.py` — мониторинг/метрики.

## Где смотреть API

- Swagger UI: `http://localhost:8000/docs`
- OpenAPI JSON: `http://localhost:8000/openapi.json`
